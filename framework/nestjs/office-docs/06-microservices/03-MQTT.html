# MQTT

MQTT（消息队列遥测传输）是一种开放源代码的轻量级消息传递协议，针对高延迟进行了优化。该协议提供了一种可扩展且经济高效的方式来使用发布/订阅模型连接设备。基于MQTT的通信系统由发布服务器，代理和一个或多个客户端组成。它设计用于受限设备和低带宽，高延迟或不可靠的网络。

# 安装

要开始构建基于MQTT的微服务，请首先安装所需的软件包：

$ npm i --save mqtt

# 概述

要使用MQTT传输器，请将以下选项对象传递给该createMicroservice()方法：

const app = await NestFactory.createMicroservice<MicroserviceOptions>(ApplicationModule, {
  transport: Transport.MQTT,
  options: {
    host: 'localhost',
    port: 1883,
  },
});

TIP

提示 该Transport枚举从导入的@nestjs/microservices包。

选项

该options对象特定于所选的运输者。该MQTT转运公开描述的特性在这里。

# 客户

与其他微服务传输器一样，您可以使用多种选项来创建MQTT ClientProxy实例。

创建实例的一种方法是使用ClientsModule。要使用创建一个客户端实例ClientsModule，请导入该实例，并使用register()方法传递一个选项对象，该对象具有与createMicroservice()方法中上面显示的相同的属性，以及用作name注入令牌的属性。ClientsModule在此处了解更多信息。

@Module({
  imports: [
    ClientsModule.register([
      {
        name: 'MATH_SERVICE',
        transport: Transport.MQTT,
        options: {
          host: 'localhost',
          port: 1883,
        }
      },
    ]),
  ]
  ...
})

也可以使用其他创建客户端的选项（ClientProxyFactory或@Client()）。您可以在这里阅读有关它们的信息。

# 上下文

在更复杂的方案中，您可能需要访问有关传入请求的更多信息。使用MQTT传输器时，可以访问该MqttContext对象。

@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: MqttContext) {
  console.log(`Topic: ${context.getTopic()}`);
}

TIP

提示 @Payload()，@Ctx()并且MqttContext从@nestjs/microservices包中导入。

要访问原始的mqtt 数据包，请使用对象的getPacket()方法MqttContext，如下所示：

@MessagePattern('notifications')
getNotifications(@Payload() data: number[], @Ctx() context: MqttContext) {
  console.log(context.getPacket());
}

# 通配符

订阅可以是显式主题，也可以包括通配符。有两个通配符，+和#。+是单级通配符，#而是涵盖许多主题级的多级通配符。

@MessagePattern('sensors/+/temperature/+')
getTemperature(@Ctx() context: MqttContext) {
  console.log(`Topic: ${context.getTopic()}`);
}
