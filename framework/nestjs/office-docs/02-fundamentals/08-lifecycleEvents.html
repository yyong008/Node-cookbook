# 生命周期事件

Nest应用程序以及每个应用程序元素都有一个由Nest管理的生命周期。鸟巢提供生命周期挂钩，让洞察关键生命周期事件，以及行动能力（运行注册代码在你的module，injectable或controller发生时）。

# 生命周期序列

下图描述了从应用程序启动到节点进程退出之间的关键应用程序生命周期事件的顺序。我们可以将整个生命周期分为三个阶段：初始化，运行和终止。使用此生命周期，您可以计划适当的模块和服务初始化，管理活动连接以及在收到终止信号时正常关闭应用程序。

# 生命周期事件

生命周期事件在应用程序引导和关闭期间发生。嵌套调用上注册的生命周期钩子方法modules，injectables并controllers在以下每个生命周期事件的（关闭挂钩需要首先使能，如所描述的下面）。如上图所示，Nest还调用适当的基础方法来开始监听连接，并停止监听连接。

|序号|生命周期挂钩方法|生命周期事件触发挂钩方法调用| |1|onModuleInit()|主机模块的依赖关系解决后调用。| |2|onApplicationBootstrap()|一旦所有模块都已初始化，但在侦听连接之前调用。 | |3|onModuleDestroy()|在SIGTERM收到终止信号（例如）之后调用。| |4|beforeApplicationShutdown()|所有onModuleDestroy()处理程序完成后调用（已解决或拒绝了承诺）； 一旦完成（承诺已解决或被拒绝），所有现有连接将被关闭（app.close()称为）。| |5|onApplicationShutdown()|连接关闭后调用（app.close()解析）。 |

# 用法

每个生命周期挂钩都由一个接口表示。接口在技术上是可选的，因为在TypeScript编译后它们不存在。尽管如此，最好还是使用它们，以便从强大的键入和编辑器工具中受益。要注册生命周期挂钩，请实现适当的接口。例如，要在特定类（例如，Controller，Provider或Module）上注册在模块初始化期间要调用OnModuleInit的onModuleInit()方法，请通过提供一种方法来实现接口，如下所示：

import { Injectable, OnModuleInit } from '@nestjs/common';

@Injectable()
export class UsersService implements OnModuleInit {
  onModuleInit() {
    console.log(`The module has been initialized.`);
  }
}

# 异步初始化

无论是OnModuleInit与OnApplicationBootstrap钩子允许你推迟应用初始化处理（返回一个Promise或标记方法async和await在该方法中体异步方法完成）。

async onModuleInit(): Promise<void> {
  await this.fetch();
}

# 关闭应用

的beforeApplicationShutdown()和onApplicationShutdown()挂钩被称为在终止阶段（响应于系统信号，诸如SIGTERM）。此功能通常与Kubernetes，Heroku或类似服务一起使用。

要使用这些挂钩，您必须激活一个监听关闭信号的监听器。

import { NestFactory } from '@nestjs/core';
import { AppModule } from './app.module';

async function bootstrap() {
  const app = await NestFactory.create(AppModule);
  // Starts listening for shutdown hooks
  app.enableShutdownHooks();
  await app.listen(3000);
}
bootstrap();

当应用程序收到终止信号时，它将调用任何已注册beforeApplicationShutdown()，然后将onApplicationShutdown()方法（以上述顺序）以相应的信号作为第一个参数。如果已注册的函数等待异步调用（返回承诺），则在承诺被解决或拒绝之前，Nest不会在序列中继续。

@Injectable()
class UsersService implements OnApplicationShutdown {
  onApplicationShutdown(signal: string) {
    console.log(signal); // e.g. "SIGINT"
  }
}
