# 订阅内容 Subscriptions

除了使用查询获取数据和使用突变来修改数据外，GraphQL规范还支持第三种操作类型，称为subscription。GraphQL订阅是一种将数据从服务器推送到选择侦听来自服务器的实时消息的客户端的方法。订阅与查询相似，因为订阅指定了要传递给客户端的一组字段，但是每次在服务器上发生特定事件时，都会打开一个通道并向客户端发送结果，而不是立即返回单个答案。

订阅的一个常见用例是将特定事件通知客户端，例如，创建新对象，更新字段等（在此处了解更多信息）。

# Subscriptions

要启用订阅，请将installSubscriptionHandlers属性设置为true。

GraphQLModule.forRoot({
  installSubscriptionHandlers: true,
}),

# 代码优先 Code first

要使用代码优先方法创建订阅，我们将使用@Subscription()装饰器和包中的PubSub类，该graphql-subscriptions包提供了简单的publish / subscribe API。

订阅处理程序负责通过调用来订阅事件PubSub#asyncIterator。此方法采用单个参数，该参数triggerName对应于事件主题名称。

const pubSub = new PubSub();

@Resolver(of => Author)
export class AuthorResolver {
  // ...
  @Subscription(returns => Comment)
  commentAdded() {
    return pubSub.asyncIterator('commentAdded');
  }
}

当前的实现还要求方法处理程序名称与匹配triggerName。也就是说，在上面的代码示例中，方法名称commentAdded()必须与triggerName传递给的方法名称匹配pubSub.asyncIterator('commentAdded')。

TIP

提示 所有装饰器均从@nestjs/graphql包中导出，而PubSub类则从graphql-subscriptions包中导出。

注意 PubSub是一个公开简单publish和的类subscribe API。在此处了解更多信息。请注意，Apollo文档警告该默认实现不适合生产（在此处了解更多信息）。生产应用程序应使用PubSub外部商店支持的实现（在此处了解更多信息）。

这将导致在SDL中生成GraphQL模式的以下部分：

type Subscription {
  commentAdded(): Comment!
}

要发布事件，请使用PubSub#publish方法。当对象图的一部分已更改时，通常在突变内使用此选项来触发客户端更新。例如：

// posts / posts.resolver.tsJS

@Mutation(returns => Post)
async addComment(
  @Args('postId', { type: () => Int }) postId: number,
  @Args('comment', { type: () => Comment }) comment: CommentInput,
) {
  const newComment = this.commentsService.addComment({ id: postId, comment });
  pubSub.publish('commentAdded', newComment);
  return newComment;
}

要过滤掉特定事件，请将filter属性设置为过滤器功能。此函数的行为类似于传递给数组的函数filter。它有两个参数：payload包含事件有效负载（由事件发布者发送），以及variables接受在订阅请求期间传递的所有参数。它返回一个布尔值，确定是否应将此事件发布给客户端侦听器。

@Subscription(returns => Comment, {
  filter: (payload, variables) =>
    payload.commentAdded.title === variables.title,
})
commentAdded(@Args('title') title: string) {
  return pubSub.asyncIterator('commentAdded');
}

要更改已发布的事件有效负载，请将resolve属性设置为一个函数。该函数接收事件有效负载（由事件发布者发送），并返回适当的值。

@Subscription(returns => Comment, {
  resolve: value => value,
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

如果您需要访问注入的提供程序（例如，使用外部服务来验证数据），请使用以下构造。

@Subscription(returns => Comment, {
  resolve(this: AuthorResolver, value) {
    // "this" refers to an instance of "AuthorResolver"
    return value;
  }
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

过滤器的结构相同：

@Subscription(returns => Comment, {
  filter(this: AuthorResolver, payload, variables) {
    // "this" refers to an instance of "AuthorResolver"
    return payload.commentAdded.title === variables.title;
  }
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

# 架构第一 => Schema first

要在Nest中创建等效订阅，我们将使用@Subscription()装饰器。

const pubSub = new PubSub();

@Resolver('Author')
export class AuthorResolver {
  // ...
  @Subscription()
  commentAdded() {
    return pubSub.asyncIterator('commentAdded');
  }
}

要根据上下文和参数过滤掉特定事件，请设置filter属性。

@Subscription('commentAdded', {
  filter: (payload, variables) =>
    payload.commentAdded.title === variables.title,
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

要更改已发布的有效负载，我们可以使用一个resolve函数。

@Subscription('commentAdded', {
  resolve: value => value,
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

如果您需要访问注入的提供程序（例如，使用外部服务来验证数据），请使用以下构造：

@Subscription('commentAdded', {
  resolve(this: AuthorResolver, value) {
    // "this" refers to an instance of "AuthorResolver"
    return value;
  }
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

过滤器的结构相同：

@Subscription('commentAdded', {
  filter(this: AuthorResolver, payload, variables) {
    // "this" refers to an instance of "AuthorResolver"
    return payload.commentAdded.title === variables.title;
  }
})
commentAdded() {
  return pubSub.asyncIterator('commentAdded');
}

最后一步是更新类型定义文件。

type Author {
  id: Int!
  firstName: String
  lastName: String
  posts: [Post]
}

type Post {
  id: Int!
  title: String
  votes: Int
}

type Query {
  author(id: Int!): Author
}

type Comment {
  id: String
  content: String
}

type Subscription {
  commentAdded(title: String!): Comment
}

这样，我们就创建了一个commentAdded(title: String!): Comment订阅。您可以在此处找到完整的示例实现。

# PubSub

我们在PubSub上面实例化了一个本地实例。首选方法是定义PubSub为提供程序，并通过构造函数将其注入（使用@Inject()装饰器）。这使我们可以在整个应用程序中重用实例。例如，如下定义提供者，然后'PUB_SUB'在需要的地方注入。

{
  provide: 'PUB_SUB',
  useValue: new PubSub(),
}

# 自定义订阅服务器

要自定义订阅服务器（例如，更改侦听器端口），请使用subscriptionsoptions属性（了解更多信息）。

GraphQLModule.forRoot({
  installSubscriptionHandlers: true,
  subscriptions: {
    keepAlive: 5000,
  }
}),
