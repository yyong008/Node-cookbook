# 任务调度 => Task Scheduling

任务计划允许您计划任意代码（方法/功能）以固定的日期/时间，重复间隔或在指定间隔后执行一次。在Linux世界中，这通常由OS级别的cron等程序包处理。对于Node.js应用程序，有几个程序包可模拟cron类功能。Nest提供了该@nestjs/schedule软件包，该软件包与流行的Node.js node-cron软件包集成在一起。我们将在本章中介绍该软件包。

# 安装 => Installation

要开始使用它，我们首先安装所需的依赖项。

npm install --save @nestjs/schedule

要激活作业调度，请将ScheduleModule导入到根目录AppModule并运行forRoot()静态方法，如下所示：

// app.module.ts

import { Module } from '@nestjs/common';
import { ScheduleModule } from '@nestjs/schedule';

@Module({
  imports: [
    ScheduleModule.forRoot()
  ],
})
export class AppModule {}

该.forRoot()调用将初始化调度程序，并注册应用程序内存在的所有声明性cron作业，超时和时间间隔。在发生onApplicationBootstrap生命周期挂钩时进行注册，以确保所有模块均已加载并声明了任何计划的作业。

# 声明式 cron 事务 => Declarative cron jobs

cron作业可安排任意功能（方法调用）以自动运行。Cron作业可以运行：

在指定的日期/时间一次。 定期进行；周期性作业可以在指定的时间间隔内（例如每小时一次，每周一次，每5分钟一次）在指定的时间运行 @Cron()在包含要执行代码的方法定义之前用装饰器声明cron作业，如下所示：

import { Injectable, Logger } from '@nestjs/common';
import { Cron } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron('45 * * * * *')
  handleCron() {
    this.logger.debug('Called when the current second is 45');
  }
}

在此示例中，handleCron()每当当前秒为时将调用该方法45。换句话说，该方法将在45秒标记处每分钟运行一次。

该@Cron()装饰支持所有标准的cron模式：

  • 星号（例如*）
  • 范围（例如1-3,5）
  • 步骤（例如*/2）

在上面的示例中，我们传递45 * * * * *给了装饰器。以下键显示了cron模式字符串中每个位置的解释方式：

* * * * * *
| | | | | |
| | | | | day of week
| | | | month
| | | day of month
| | hour
| minute
second (optional)

一些样本cron模式是：

* * * * * *

每一秒

45 * * * * *

每分钟，在第45秒

* 10 * * * *

每小时，在第10分钟开始时

0 */30 9-17 * * *

上午9点至下午5点之间每30分钟一班

0 30 11 * * 1-5

周一至周五上午11:30

该@nestjs/schedule软件包为常用的cron模式提供了一个方便的枚举。您可以按如下方式使用此枚举：

import { Injectable, Logger } from '@nestjs/common';
import { Cron, CronExpression } from '@nestjs/schedule';

@Injectable()
export class TasksService {
  private readonly logger = new Logger(TasksService.name);

  @Cron(CronExpression.EVERY_45_SECONDS)
  handleCron() {
    this.logger.debug('Called every 45 seconds');
  }
}

在此示例中，该handleCron()方法将每秒钟调用一次45。

或者，您可以Date向@Cron()装饰器提供JavaScript 对象。这样做会使作业在指定的日期准确执行一次。

TIP

提示 使用JavaScript日期算术来计划相对于当前日期的作业。例如，@Cron(new Date(Date.now() + 10 * 1000))计划作业在应用程序启动后10秒钟运行。

您可以在声明cron作业后访问和控制它，或者使用Dynamic API动态创建cron作业（在运行时定义其cron模式）。要通过API访问声明性cron作业，您必须通过将name属性传递到可选的options对象中作为装饰器的第二个参数来将作业与名称相关联，如下所示：

@Cron('* * 8 * * *', {
  name: 'notifications',
})
triggerNotifications() {}

# 声明间隔 => Declarative intervals

要声明某个方法应以（重复）指定的间隔运行，请在方法定义的@Interval()前面加上装饰器。将间隔值（以毫秒为单位）传递给装饰器，如下所示：

@Interval(10000)
handleInterval() {
  this.logger.debug('Called every 10 seconds');
}

TIP

提示 此机制在 后台 使用JavaScript setInterval()函数。您还可以利用Cron作业安排重复作业。

如果要通过Dynamic API从声明类的外部控制声明间隔，请使用以下构造将间隔与名称关联：

@Interval('notifications', 2500)
handleInterval() {}

动态API还允许创建动态的间隔，其中所述间隔的属性是在运行时定义，列表和删除它们。

# 声明超时 => Declarative timeouts

要声明某个方法应在指定的超时（一次）运行，请在方法定义的@Timeout()前面加上装饰器。从应用程序启动传递相对时间偏移（以毫秒为单位）到装饰器，如下所示：

@Timeout(5000)
handleTimeout() {
  this.logger.debug('Called once after 5 seconds');
}

TIP

提示 此机制在 后台 使用JavaScript setTimeout()函数。

如果要通过Dynamic API从声明类的外部控制声明超时，请使用以下构造将超时与名称相关联：

@Timeout('notifications', 2500)
handleTimeout() {}

使用动态API，还可以创建动态超时，并在运行时定义超时属性，并列出和删除它们。

# 动态计划模块API => Dynamic schedule module API

该@nestjs/schedule模块提供了动态API，可用于管理声明性cron作业，超时和间隔。该API还支持创建和管理动态 cron作业，超时和间隔，这些属性是在运行时定义的。

# 动态Cron工作 => Dynamic cron jobs

CronJob使用SchedulerRegistryAPI 从代码的任何位置按名称获取对实例的引用。首先，SchedulerRegistry使用标准构造函数注入进行注入：

constructor(private schedulerRegistry: SchedulerRegistry) {}

TIP

提示 SchedulerRegistry从@nestjs/schedule包中导入。

然后在类中使用它，如下所示。假设使用以下声明创建了cron作业：

TIP

@Cron('* * 8 * * *', { name: 'notifications', }) triggerNotifications() {}

使用以下命令访问此作业：

const job = this.schedulerRegistry.getCronJob('notifications');

job.stop();
console.log(job.lastDate());

# 动态间隔 => Dynamic intervals

该getCronJob()方法返回命名的cron作业。返回的CronJob对象具有以下方法：

  • stop() -停止计划运行的作业。
  • start() -重新启动已停止的作业。
  • setTime(time: CronTime) -停止工作，为其设置新的时间，然后开始工作
  • lastDate() -返回工作最后执行日期的字符串表示
  • nextDates(count: number)-返回表示即将执行的作业日期count的moment对象数组（大小）。

TIP

提示 toDate()在moment对象上使用以使其易于阅读的形式呈现它们。

使用该SchedulerRegistry.addCronJob()方法动态创建一个新的cron作业，如下所示：

addCronJob(name: string, seconds: string) {
  const job = new CronJob(`${seconds} * * * * *`, () => {
    this.logger.warn(`time (${seconds}) for job ${name} to run!`);
  });

  this.scheduler.addCronJob(name, job);
  job.start();

  this.logger.warn(
    `job ${name} added for each minute at ${seconds} seconds!`,
  );
}

在此代码中，我们使用包中的CronJob对象cron创建cron作业。所述CronJob构造函数采用一个cron图案（就像@Cron()装饰）作为它的第一个参数，和将要执行的回调的cron定时器触发作为第二个参数时。该SchedulerRegistry.addCronJob()方法带有两个参数：的名称CronJob和CronJob对象本身。

TIP

警告 记住SchedulerRegistry在访问之前注入。CronJob从cron包中导入。

使用以下SchedulerRegistry.deleteCronJob()方法删除命名的cron作业：

deleteCron(name: string) {
  this.scheduler.deleteCronJob(name);
  this.logger.warn(`job ${name} deleted!`);
}

使用以下SchedulerRegistry.getCronJobs()方法列出所有cron作业：

getCrons() {
  const jobs = this.scheduler.getCronJobs();
  jobs.forEach((value, key, map) => {
    let next;
    try {
      next = value.nextDates().toDate();
    } catch (e) {
      next = 'error: next fire date is in the past!';
    }
    this.logger.log(`job: ${key} -> next: ${next}`);
  });
}

该getCronJobs()方法返回一个map。在这段代码中，我们遍历地图并尝试访问nextDates()each 的方法CronJob。在CronJobAPI中，如果作业已经被触发并且没有未来的触发日期，它将引发异常。

使用该SchedulerRegistry.getInterval()方法获取对间隔的引用。如上所述，SchedulerRegistry使用标准构造函数注入进行注入：

constructor(private schedulerRegistry: SchedulerRegistry) {}

并按如下所示使用它：

const interval = this.schedulerRegistry.getInterval('notifications');
clearInterval(interval);

使用该SchedulerRegistry.addInterval()方法动态创建一个新间隔，如下所示：

addInterval(name: string, seconds: string) {
  const callback = () => {
    this.logger.warn(`Interval ${name} executing at time (${seconds})!`);
  };

  const interval = setInterval(callback, seconds);
  this.scheduler.addInterval(name, interval);
}

在这段代码中，我们创建一个标准的JavaScript间隔，然后将其传递给该ScheduleRegistry.addInterval()方法。该方法有两个参数：时间间隔的名称和时间间隔本身。

使用以下SchedulerRegistry.deleteInterval()方法删除命名间隔：

deleteInterval(name: string) {
  this.scheduler.deleteInterval(name);
  this.logger.warn(`Interval ${name} deleted!`);
}

使用以下SchedulerRegistry.getIntervals()方法列出所有间隔：

getIntervals() {
  const intervals = this.scheduler.getIntervals();
  intervals.forEach(key => this.logger.log(`Interval: ${key}`));
}
