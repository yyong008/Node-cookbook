# 记录仪 => logger

Nest带有一个内置的基于文本的记录器，该记录器可在应用程序引导和其他几种情况下使用，例如显示捕获的异常（即系统日志记录）。此功能通过包中的Logger类提供@nestjs/common。您可以完全控制日志记录系统的行为，包括以下任何一项：

完全禁用日志记录 指定详细的日志级别（例如，显示错误，警告，调试信息等） 完全覆盖默认记录器 通过扩展自定义默认记录器 利用依赖注入来简化应用程序的组成和测试 您还可以使用内置记录器，或创建自己的自定义实现，以记录自己的应用程序级事件和消息。

要获得更高级的日志记录功能，可以使用任何Node.js日志记录包（例如Winston）来实现完全定制的生产级日志记录系统。

# 基本定制 => Basic customization

要禁用日志记录，请在（作为可选）Nest应用程序选项对象中将该logger属性设置false为作为方法的第二个参数传递NestFactory.create()。

const app = await NestFactory.create(ApplicationModule, {
  logger: false,
});
await app.listen(3000);

要启用特定的日志记录级别，请将logger属性设置为字符串数组，指定要显示的日志级别，如下所示：

const app = await NestFactory.create(ApplicationModule, {
  logger: ['error', 'warn'],
});
await app.listen(3000);

阵列中的值可以是任意组合'log'，'error'，'warn'，'debug'，和'verbose'。

# 自定义实现 => Custom implementation

您可以通过将logger属性的值设置为满足LoggerService接口的对象，来提供供Nest用于系统日志记录的自定义日志记录器实现。例如，您可以告诉Nest使用内置的全局JavaScript console对象（实现LoggerService接口），如下所示：

const app = await NestFactory.create(ApplicationModule, {
  logger: console,
});
await app.listen(3000);

实现自己的自定义记录器非常简单。只需实现LoggerService接口的每种方法，如下所示。

import { LoggerService } from '@nestjs/common';

export class MyLogger implements LoggerService {
  log(message: string) {
    /* your implementation */
  }
  error(message: string, trace: string) {
    /* your implementation */
  }
  warn(message: string) {
    /* your implementation */
  }
  debug(message: string) {
    /* your implementation */
  }
  verbose(message: string) {
    /* your implementation */
  }
}

然后，您可以MyLogger通过loggerNest应用程序选项对象的属性提供的实例。

const app = await NestFactory.create(ApplicationModule, {
  logger: new MyLogger(),
});
await app.listen(3000);

这种技术虽然简单，但并未为MyLogger该类利用依赖注入。这可能会带来一些挑战，尤其是对于测试而言，并限制的可重用性MyLogger。有关更好的解决方案，请参见下面的“ 依赖项注入”部分。

# 扩展内置记录仪 => Extend build-in logger

与其从头编写记录器，不如通过扩展内置Logger类并覆盖默认实现的选定行为，可以满足您的需求。

import { Logger } from '@nestjs/common';

export class MyLogger extends Logger {
  error(message: string, trace: string) {
    // add your tailored logic here
    super.error(message, trace);
  }
}

您可以在功能模块中使用这样的扩展记录器，如下面的“ 使用记录器进行应用程序记录”部分所述。

您可以通过logger应用程序选项对象的属性（如上面的“ 自定义实现”部分中所示）或通过使用下面的“ 依赖项注入”部分中所示的技术来传递它的实例，来告诉Nest将扩展记录器用于系统日志记录。如果这样做，则应小心调用super，如上面的示例代码所示，将特定的log方法调用委派给父（内置）类，以便Nest可以依赖其期望的内置功能。

# 依赖注入 => Dependency injection

对于更高级的日志记录功能，您将需要利用依赖注入。例如，您可能想将a注入ConfigService到记录器中以对其进行自定义，然后将您的自定义记录器注入到其他控制器和/或提供程序中。要为您的自定义记录器启用依赖项注入，请创建一个实现LoggerService并在某个模块中将其注册为提供程序的类。例如，您可以

  1. 定义一个MyLogger类，以扩展内置Logger或完全覆盖内置，如前几节所示。
  2. 创建LoggerModule如下所示的，然后MyLogger从该模块提供。
import { Module } from '@nestjs/common';
import { MyLogger } from './my-logger.service';

@Module({
  providers: [MyLogger],
  exports: [MyLogger],
})
export class LoggerModule {}

通过这种结构，您现在可以提供供其他模块使用的自定义记录器。由于您的MyLogger类是模块的一部分，因此可以使用依赖项注入（例如注入ConfigService）。提供此自定义记录器以供Nest用于系统记录（例如，用于引导和错误处理）还需要另一种技术。

因为应用程序实例化（NestFactory.create()）发生在任何模块的上下文之外，所以它不参与初始化的常规依赖注入阶段。因此，我们必须确保至少有一个应用程序模块导入，LoggerModule以触​​发Nest实例化我们MyLogger类的单例实例。然后，我们可以指示Nest使用MyLogger以下结构的相同单例实例：

const app = await NestFactory.create(ApplicationModule, {
  logger: false,
});
app.useLogger(app.get(MyLogger));
await app.listen(3000);

在这里，我们get()在NestApplication实例上使用方法来检索MyLogger对象的单例实例。这项技术本质上是一种“插入”记录器实例以供Nest使用的方法。该app.get()调用将检索的一个实例，MyLogger并依赖于该实例被另一个模块中第一注入，如上所述。

您也可以MyLogger在要素类中注入此提供程序，从而确保Nest系统日志记录和应用程序日志记录之间的记录行为一致。有关更多信息，请参见下面的使用记录器记录应用程序。

# 使用应用程序日志记录器 => Using the logging

我们可以结合以上几种技术来在Nest系统日志记录和我们自己的应用程序事件/消息日志记录之间提供一致的行为和格式。在本节中，我们将通过以下步骤来实现：

  1. 我们扩展了内置的记录器并自定义context日志消息的一部分（例如，NestFactory如下所示的日志行中方括号中的短语）。
[Nest] 19096   - 12/08/2019, 7:12:59 AM   [NestFactory] Starting Nest application...
  1. 我们将的瞬时实例注入Logger到我们的功能模块中，以便每个模块都有自己的自定义上下文。
  2. 我们为Nest提供此扩展记录器，以用于系统记录。

首先，使用以下代码扩展内置记录器。我们提供该scope选项作为Logger类的配置元数据，并指定一个临时范围，以确保我们Logger在每个功能模块中都有一个唯一的实例。在这个例子中，我们没有延长个体Logger的方法（如log()，warn()等），您也可以选择这样做。

import { Injectable, Scope, Logger } from '@nestjs/common';

@Injectable({ scope: Scope.TRANSIENT })
export class MyLogger extends Logger {}

接下来，LoggerModule使用以下结构创建一个：

import { Module } from '@nestjs/common';
import { MyLogger } from './my-logger.service';

@Module({
  providers: [MyLogger],
  exports: [MyLogger],
})
export class LoggerModule {}

接下来，将导入LoggerModule到功能模块中。然后设置记录器上下文，并开始使用上下文感知的自定义记录器，如下所示：

import { Injectable } from '@nestjs/common';
import { MyLogger } from './my-logger.service';

@Injectable()
export class CatsService {
  private readonly cats: Cat[] = [];

  constructor(private myLogger: MyLogger) {
    this.myLogger.setContext('CatsService');
  }

  findAll(): Cat[] {
    this.myLogger.warn('About to return cats!');
    return this.cats;
  }
}

最后，指示Nest在main.ts文件中使用自定义记录器的实例，如下所示。当然，在这个例子中，我们还没有实际的定制记录器行为（通过延伸Logger等方法log()，warn()等），所以实际上不需要此步骤。但是，如果您向这些方法中添加了自定义逻辑并希望Nest使用相同的实现，则将是需要的。

const app = await NestFactory.create(ApplicationModule, {
  logger: false,
});
app.useLogger(new MyLogger());
await app.listen(3000);

# 使用额外的记录仪 => Use External logger

生产应用程序通常具有特定的日志记录要求，包括高级过滤，格式化和集中式日志记录。Nest的内置记录器用于监视Nest系统的行为，并且对于开发中的功能模块中的基本格式化文本记录也很有用，但是生产应用程序通常会利用Winston等专用记录模块。与任何标准Node.js应用程序一样，您可以在Nest中充分利用此类模块。
