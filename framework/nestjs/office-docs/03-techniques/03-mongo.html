# Mongo

Nest支持两种与MongoDB数据库集成的方法。您可以使用此处描述的内置TypeORM模块（该模块具有用于MongoDB的连接器），也可以使用Mongoose（最受欢迎的MongoDB对象建模工具）。在本章中，我们将使用专用软件包描述后者。@nestjs/mongoose

首先安装所需的依赖项：

$ npm install --save @nestjs/mongoose mongoose

安装过程完成后，我们可以将导入MongooseModule到根目录中AppModule。

// app.module.ts

import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [MongooseModule.forRoot('mongodb://localhost/nest')],
})
export class AppModule {}

该forRoot()方法接受相同的配置对象作为mongoose.connect()从 mongoose 包，如所描述这里。

# 对象关系映射迁移 => TypeORM Intergration

对于Mongoose，一切都源自Schema。让我们定义CatSchema：

// schemas / cat.schema.ts

import * as mongoose from 'mongoose';

export const CatSchema = new mongoose.Schema({
  name: String,
  age: Number,
  breed: String,
});

该cat.schema文件位于cats目录中的文件夹中，我们还在其中定义CatsModule。尽管可以将模式文件存储在任意位置，但我们建议将它们存储在相应模块目录中与它们相关的域对象附近。

让我们看一下CatsModule：

// cats.module.ts

import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';
import { CatsController } from './cats.controller';
import { CatsService } from './cats.service';
import { CatSchema } from './schemas/cat.schema';

@Module({
  imports: [MongooseModule.forFeature([{ name: 'Cat', schema: CatSchema }])],
  controllers: [CatsController],
  providers: [CatsService],
})
export class CatsModule {}

在MongooseModule提供了forFeature()将模块配置，其中包括模型应在当前范围内进行注册限定方法。如果您还想在另一个模块中使用模型，请将MongooseModule添加到的exports部分中，CatsModule然后导入CatsModule另一个模块。

一旦注册了架构，就可以使用装饰器将Cat模型注入：CatsService@InjectModel()

// cats.service.ts

import { Model } from 'mongoose';
import { Injectable } from '@nestjs/common';
import { InjectModel } from '@nestjs/mongoose';
import { Cat } from './interfaces/cat.interface';
import { CreateCatDto } from './dto/create-cat.dto';

@Injectable()
export class CatsService {
  constructor(@InjectModel('Cat') private catModel: Model<Cat>) {}

  async create(createCatDto: CreateCatDto): Promise<Cat> {
    const createdCat = new this.catModel(createCatDto);
    return createdCat.save();
  }

  async findAll(): Promise<Cat[]> {
    return this.catModel.find().exec();
  }
}

# 连接

有时您可能需要访问本地Mongoose Connection对象。例如，您可能想在连接对象上进行本机API调用。您可以通过使用@InjectConnection()装饰器来注入Mongoose Connection，如下所示：

import { Injectable } from '@nestjs/common';
import { InjectConnection } from '@nestjs/mongoose';
import { Connection } from 'mongoose';

@Injectable()
export class CatsService {
  constructor(@InjectConnection() private connection: Connection) {}
}

# Multiple databases

一些项目需要多个数据库连接。这也可以通过该模块来实现。要使用多个连接，请首先创建连接。在这种情况下，连接命名成为强制性的。

// app.module.ts

import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [
    MongooseModule.forRoot('mongodb://localhost/test', {
      connectionName: 'cats',
    }),
    MongooseModule.forRoot('mongodb://localhost/users', {
      connectionName: 'users',
    }),
  ],
})
export class AppModule {}

TIP

注意 请注意，您不应有多个没有名称或具有相同名称的连接，否则它们将被覆盖。

使用此设置，您必须告诉MongooseModule.forFeature()功能应使用哪个连接。

@Module({
  imports: [
    MongooseModule.forFeature([{ name: 'Cat', schema: CatSchema }], 'cats'),
  ],
})
export class AppModule {}

您还可以为Connection给定的连接注入：

import { Injectable } from '@nestjs/common';
import { InjectConnection } from '@nestjs/mongoose';
import { Connection } from 'mongoose';

@Injectable()
export class CatsService {
  constructor(@InjectConnection('cats') private connection: Connection) {}
}

# 中间件（Hooks）

中间件（也称为前挂钩和后挂钩）是在异步功能执行期间通过控制传递的功能。中间件在模式级别上指定，对于编写插件（源）很有用。调用模型pre()或post()在模型编译后无法在Mongoose中使用。要在模型注册之前注册钩子，请使用的forFeatureAsync()方法和MongooseModule工厂提供者（即useFactory）一起使用。使用这种技术，您可以访问模式对象，然后使用pre()或post()方法在该模式上注册一个钩子。请参见下面的示例：

@Module({
  imports: [
    MogooseModule.forFeatureAsync([
      {
        name: 'Cat',
        useFactory: () => {
          const schema = CatsShema;
          schema.pre('save', () = console.log('Hello from pre save')
          return schema;
        }
      }
    ])
  ]
})
export class AppModule {}

像其他工厂提供者一样，我们的工厂功能可以async并且可以通过注入依赖inject。

@Module({
  imports: [
    MongooseModule.forFeatureAsync([
      {
        name: 'Cat',
        imports: [ConfigModule],
        useFactory: (configService: ConfigService) => {
          const schema = CatsSchema;
          schema.pre('save', () =>
            console.log(
              `${configService.getString('APP_NAME')}: Hello from pre save`,
            ),
          );
          return schema;
        },
        inject: [ConfigService],
      },
    ]),
  ],
})
export class AppModule {}

# 插件

要为给定架构注册插件，请使用forFeatureAsync()方法。

@Module({
  imports: [
    MongooseModule.forFeatureAsync([
      {
        name: 'Cat',
        useFactory: () => {
          const schema = CatsSchema;
          schema.plugin(require('mongoose-autopopulate'));
          return schema;
        },
      },
    ]),
  ],
})
export class AppModule {}

要一次为所有模式注册插件，请调用对象的.plugin()方法Connection。在创建模型之前，您应该访问连接；为此，请使用connectionFactory：

app.module.tsJS

import { Module } from '@nestjs/common';
import { MongooseModule } from '@nestjs/mongoose';

@Module({
  imports: [
    MongooseModule.forRoot('mongodb://localhost/test', {
      connectionFactory: (connection) => {
        connection.plugin(require('mongoose-autopopulate'));
        return connection;
      }
    }),
  ],
})
export class AppModule {}

# 测试

在对应用程序进行单元测试时，我们通常希望避免任何数据库连接，从而使我们的测试套件更易于设置和执行。但是我们的类可能取决于从连接实例中提取的模型。我们如何解决这些问题？解决方案是创建模拟模型。

为了简化操作，@nestjs/mongoose程序包公开了一个getModelToken()函数，该函数根据令牌名称返回准备好的注入令牌。使用此令牌，你可以很容易地提供使用任何标准的模拟实现自定义提供技术，包括useClass，useValue，和useFactory。例如：

@Module({
  providers: [
    CatsService,
    {
      provide: getModelToken('Cat'),
      useValue: catModel,
    },
  ],
})
export class CatsModule {}

在此示例中，catModel每当任何消费者Model使用@InjectModel()装饰器注入时，都将提供硬编码（对象实例）。

# 异步配置

当您需要异步而不是静态地传递模块选项时，请使用该forRootAsync()方法。与大多数动态模块一样，Nest提供了多种技术来处理异步配置。

一种技术是使用工厂功能：

MongooseModule.forRootAsync({
  useFactory: () => ({
    uri: 'mongodb://localhost/nest',
  }),
});

像其他工厂提供者一样，我们的工厂功能可以async并且可以通过注入依赖inject。

MongooseModule.forRootAsync({
  imports: [ConfigModule],
  useFactory: async (configService: ConfigService) => ({
    uri: configService.getString('MONGODB_URI'),
  }),
  inject: [ConfigService],
});

或者，您可以MongooseModule使用类而不是工厂来配置，如下所示：

MongooseModule.forRootAsync({
  useClass: MongooseConfigService,
});

上面的构造在MongooseConfigService内部实例化MongooseModule，使用它来创建所需的options对象。请注意，在此示例中，MongooseConfigService必须实现MongooseOptionsFactory接口，如下所示。在MongooseModule将调用createMongooseOptions()所提供的类的实例化的对象上的方法。

@Injectable()
class MongooseConfigService implements MongooseOptionsFactory {
  createMongooseOptions(): MongooseModuleOptions {
    return {
      uri: 'mongodb://localhost/nest',
    };
  }
}

如果要重用现有的选项提供程序，而不是在中创建私有副本MongooseModule，请使用useExisting语法。

MongooseModule.forRootAsync({
  imports: [ConfigModule],
  useExisting: ConfigService,
});
