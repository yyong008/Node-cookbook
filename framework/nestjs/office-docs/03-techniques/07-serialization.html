# 序列化

序列化是在网络响应中返回对象之前发生的过程。这是提供用于转换和清理要返回给客户端的数据的规则的适当位置。例如，应始终将敏感数据（例如密码）从响应中排除。或者，某些属性可能需要进行其他转换，例如仅发送实体属性的子集。手动执行这些转换可能很乏味且容易出错，并且可能使您不确定是否已涵盖所有情况。

# 概述

Nest提供了内置功能，可帮助确保可以直接方式执行这些操作。该ClassSerializerInterceptor拦截器使用功能强大的类变压器包提供变换对象的声明和可扩展的方式。它执行的基本操作是获取方法处理程序返回的值，并应用class-transformer中的classToPlain()函数。这样，它可以将装饰器表示的规则应用于实体/ DTO类，如下所述。class-transformer

# 概述

Nest提供了内置功能，可帮助确保可以直接方式执行这些操作。该ClassSerializerInterceptor拦截器使用功能强大的类变压器包提供变换对象的声明和可扩展的方式。它执行的基本操作是获取方法处理程序返回的值，并应用class-transformer中的classToPlain()函数。这样，它可以将装饰器表示的规则应用于实体/ DTO类，如下所述。class-transformer

import { Exclude } from 'class-transformer';

export class UserEntity {
  id: number;
  firstName: string;
  lastName: string;

  @Exclude()
  password: string;

  constructor(partial: Partial<UserEntity>) {
    Object.assign(this, partial);
  }
}

现在考虑一个带有方法处理程序的控制器，该方法返回该类的实例。

@UseInterceptors(ClassSerializerInterceptor)
@Get()
findOne(): UserEntity {
  return new UserEntity({
    id: 1,
    firstName: 'Kamil',
    lastName: 'Mysliwiec',
    password: 'password',
  });
}

WARNING

警告 注意，我们必须返回该类的实例。例如，如果您返回一个普通的JavaScript对象{ user: new UserEntity() }，则该对象将无法正确序列化。

TIP

提示 ClassSerializerInterceptor是进口@nestjs/common

当请求此终结点时，客户端收到以下响应：

{
  "id": 1,
  "firstName": "Kamil",
  "lastName": "Mysliwiec"
}

请注意，拦截器可以在整个应用程序范围内应用（如此处所述）。拦截器和实体类声明的组合可确保任何返回a的方法UserEntity都将确保删除该password属性。这使您可以度量该业务规则的集中实施。

# 公开属性

您可以使用@Expose()装饰器为属性提供别名，或执行一个函数以计算属性值（类似于getter函数），如下所示。

@Expose()
get fullName(): string {
  return `${this.firstName} ${this.lastName}`;
}

# 转换

您可以使用@Transform()装饰器执行其他数据转换。例如，以下构造返回的名称属性，RoleEntity而不是返回整个对象。

@Transform(role => role.name)
role: RoleEntity;

# 选项传递

您可能需要修改转换函数的默认行为。要覆盖默认设置，请options使用@SerializeOptions()装饰器将其传递给对象。

@SerializeOptions({
  excludePrefixes: ['_'],
})
@Get()
findOne(): UserEntity {
  return new UserEntity();
}

TIP

提示 该@SerializeOptions()装饰采用进口@nestjs/common。

传递的选项@SerializeOptions()作为基础classToPlain()函数的第二个参数传递。在此示例中，我们将自动排除所有以_前缀开头的属性。

# Web 套接字和微服务

尽管本章显示了使用HTTP样式应用程序（例如Express或Fastify）的示例ClassSerializerInterceptor，但无论使用哪种传输方法，WebSockets和微服务的工作原理都是相同的。
