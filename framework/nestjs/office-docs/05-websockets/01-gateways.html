# 网关

本文档中其他地方讨论的大多数概念，例如依赖项注入，装饰器，异常过滤器，管道，防护和拦截器，同样适用于网关。Nest会尽可能地抽象化实现细节，以便相同的组件可以在基于HTTP的平台，WebSocket和微服务上运行。本节涵盖特定于WebSocket的Nest方面。

在Nest中，网关只是带有@WebSocketGateway()装饰器的类。从技术上讲，网关与平台无关，一旦创建适配器，它便与任何WebSockets库兼容。有两种现成的WS平台支持：socket.io和ws。您可以选择最适合您的需求。另外，您可以按照本指南构建自己的适配器。

TIP

提示 网关可以被视为提供者 ; 这意味着他们可以通过类构造函数注入依赖项。同样，网关也可以由其他类（提供者和控制器）注入。

# 安装

要开始构建基于WebSockets的应用程序，请首先安装所需的软件包：

$ npm i --save @nestjs/websockets @nestjs/platform-socket.io
$ npm i --save-dev @types/socket.io

通常，除非您的应用程序不是Web应用程序，或者您手动更改了端口，否则每个网关都在与HTTP服务器侦听相同的端口上。可以通过将参数传递给@WebSocketGateway(80)装饰器（其中80是选定的端口号）来修改此默认行为。您还可以使用以下结构来设置网关使用的名称空间：

@WebSocketGateway(80, { namespace: 'events' })

TIP

警告 网关直到在现有模块的providers数组中被引用之前，才实例化。

您可以将第二个参数传递给装饰器，将任何受支持的选项传递给套接字构造@WebSocketGateway()器，如下所示：

@WebSocketGateway(81, { transports: ['websocket'] })

网关现在正在侦听，但我们尚未订阅任何传入消息。让我们创建一个处理程序，该处理程序将订阅events消息并使用完全相同的数据响应用户。

// events.gateway.ts

@SubscribeMessage('events')
handleEvent(@MessageBody() data: string): string {
  return data;
}

TIP

提示 @SubscribeMessage()和@MessageBody()装饰器是从@nestjs/websockets包中导入的。

如果您不想使用装饰器，则以下代码在功能上等效：

// events.gateway.ts

@SubscribeMessage('events')
handleEvent(client: Socket, data: string): string {
  return data;
}

在上面的示例中，该handleEvent()函数接受两个参数。第一个是特定于平台的套接字实例，而第二个是从客户端接收的数据。但是不建议使用此方法，因为它需要socket在每个单元测试中模拟实例。

一旦events接收到消息，则处理程序发送与已通过网络发送相同的数据的确认。此外，还可以使用特定于库的方法来发出消息，例如通过使用client.emit()方法。为了访问连接的套接字实例，请使用@ConnectedSocket()装饰器。

// events.gateway.ts

@SubscribeMessage('events')
handleEvent(
  @MessageBody() data: string,
  @ConnectedSocket() client: Socket,
): string {
  return data;
}

TIP

提示 @ConnectedSocket()装饰器是从@nestjs/websockets包中导入的。

但是，在这种情况下，您将无法利用拦截器。如果您不想回应用户，则可以简单地跳过该return语句（或显式地返回“ falsy”值，例如undefined）。

现在，当客户端发出如下消息时：

socket.emit('events', { name: 'Nest' });

该handleEvent()方法将被执行。为了侦听从上述处理程序中发出的消息，客户端必须附加一个相应的确认侦听器：

socket.emit('events', { name: 'Nest' }, data => console.log(data));

# 多重响应

确认仅发送一次。此外，本机WebSockets实现不支持它。要解决此限制，您可以返回一个包含两个属性的对象。在event其为所发射的事件的名称和data具有要被转发到客户端。

// events.gateway.ts

@SubscribeMessage('events')
handleEvent(@MessageBody() data: unknown): WsResponse<unknown> {
  const event = 'events';
  return { event, data };
}

TIP

提示 该WsResponse接口是从@nestjs/websockets包中导入的。

为了侦听传入的响应，客户端必须应用另一个事件侦听器。

socket.on('events', data => console.log(data));

# 异步响应

消息处理程序能够同步或异步响应。因此，async支持方法。消息处理程序还可以返回 Observable，在这种情况下，将发射结果值，直到流完成为止。

// events.gateway.ts

@SubscribeMessage('events')
onEvent(@MessageBody() data: unknown): Observable<WsResponse<number>> {
  const event = 'events';
  const response = [1, 2, 3];

  return from(response).pipe(
    map(data => ({ event, data })),
  );
}

在上面的示例中，消息处理程序将响应3次（数组中的每个项）。

# 生命周期挂钩

有3个有用的生命周期挂钩。它们都有对应的接口，并在下表中进行了描述：

name 说明
OnGatewayInit 强制执行该afterInit()方法。将特定于库的服务器实例作为参数（并在需要时扩展其余部分）。
OnGatewayConnection 强制执行该handleConnection()方法。将特定于库的客户端套接字实例作为参数。
OnGatewayDisconnect 强制执行该handleDisconnect()方法。将特定于库的客户端套接字实例作为参数。

TIP

提示 每个生命周期接口都从@nestjs/websockets包中公开。

# 服务器

有时，您可能希望直接访问特定于平台的本机服务器实例。对该对象的引用作为参数传递给afterInit()方法（OnGatewayInit接口）。另一种选择是使用@WebSocketServer()装饰器。

@WebSocketServer()
server: Server;

TIP

通知 该@WebSocketServer()装饰器从导入的@nestjs/websockets包。

准备好使用时，Nest会自动将服务器实例分配给该属性。
